The research behind
FrameWorkers

FrameWorkers is a dynamic, workspace-grounded multi-agent framework for open-ended AIGC video production. Instead of a fixed pipeline, a trained Director orchestrates a catalog of specialist agents over a shared workspace — planning, executing, checking, and re-planning until a film is done.

Why not a fixed pipeline

Real video-making is a multi-step workflow — scripting, asset management, cross-shot consistency — and different inputs call for different workflows. A hand-wired pipeline is brittle, and a zero-shot LLM struggles with long-horizon planning and multimodal asset routing. FrameWorkers instead treats video creation as dynamic task management.

Architecture

Strategic orchestration and concrete execution are split into complementary modules.

Director & Dynamic Task Stack

The orchestrator. It reads the brief, the agent descriptors, and the current state, then edits a dynamic task stack — adding, reordering, and revising future tasks. It plans per request and re-plans on failure, never a hard-coded graph; completed work stays as an immutable, auditable history.

Assistant & Workspace

The execution runtime. It resolves each task's inputs, invokes the chosen agent, validates the output, and persists everything to a shared workspace — files, generated assets, global memory, and logs. A “strict-out, loose-in” contract keeps agents decoupled and recoverable.

Sub-agent catalog

Capability as a catalog, not a graph. Each agent is a self-contained module described by its inputs, outputs, and trigger conditions. Routing is semantic, so new capabilities are added by registering a descriptor — no workflow redesign.

A closed production loop

Every task flows through the same plan → execute → check → re-plan cycle.

1

Plan

The Director edits the task stack from the intent, agent descriptors, and current state.

2

Execute

The Assistant resolves inputs from the workspace and invokes the right specialist agent.

3

Check

Outputs are parsed, validated, and written back as assets, memory, and logs.

4

Re-plan

On failure, the Director revises the remaining tasks — recover, don't restart.

A director that's trained, not just prompted

Zero-shot LLMs misjudge task order and skip necessary steps on long workflows. We fine-tune the Director — with supervised and reinforcement learning over simulated and dynamically generated execution data — so it selects the right agents, orders them by dependency, and builds robust execution chains that recover from failures.
